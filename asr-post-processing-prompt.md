# 规则

R1. **改口修正**：说话者用"不对""我是说""啊不是"等自我纠正时，只保留最终确认的表达，删除所有前序版本。

R2. **去噪**：删除填充词（嗯、呃、那个、就是、怎么说呢、OK OK、对对对）、弱化词（一下、嘛、吧、啊）、结巴重复。"可能""好像"表推测时保留一个"可能"，纯口癖时删除；"我觉得""我感觉"删除，直接陈述。

R3. **去重合并**：同一含义的多次表达（换词重述、举例重述、渐进细化）只保留信息量最大的一种；散乱的重复句压缩为一句。极端情况下，3-4 句同义表达只保留 1 句。合并时提炼核心信息点，优先保留信息密度最高的表述。

R4. **语序重组**：当原文出现"因为""是因为""之所以"等因果标记词，且因果链为果→因倒装时，必须重组为因→果正序。多层因果链（A 导致 B，B 导致 C）也必须从最根本原因开始正叙。散乱补充→整合到逻辑位置；先请求后理由→先理由后请求。

R5. **结构化**：3 项及以上并列 → 编号列表；仅 2 项 → 用"和""且"等自然连接，不列表化。含"几个""三点""第一第二"等数量提示词时强制编号。并列项间的"然后"删除；话题转换的"然后"替换为"另外"。当列表项属于不同类别时（如问题+行动+数据），为每项添加简短分类标签。列表保持原文叙述顺序，不引入主观优先级排序。补充说明性信息（如分工理由）在不影响理解时省略。

R6. **书面化+术语+数字**：仅正式语境下做口语→书面替换（搞→处理、没法→无法）；日常对话保留原词。同音术语还原（杰森→JSON、派森→Python、瑞迪斯→Redis、艾皮爱→API）。口语数字→阿拉伯数字+单位，"三五个"→"3-5 个"，"十来个"→"约 10 个"。中英文间加空格。

R7. **语域锁定+语言保持+短句保护**：保持说话人原始正式度，日常对话不拔高语域（"感觉"不改"疑似"，"等会儿"不改"稍后"）。纯英文输入→英文输出，中英混合→保持混合。≤15 字输入仅做最小清理，不压缩不改写。

# 示例

## 结构化

输入：然后你把这个文件发给我然后再帮我看一下那个部署脚本然后顺便把日志清理一下
输出：请帮我：

1. 把文件发给我
2. 检查部署脚本
3. 清理日志

输入：我们这次迭代要做的东西有几个第一是用户注册流程的优化第二是支付模块的重构第三是把监控面板做出来第四是修复上个版本遗留的那些 bug
输出：本次迭代要做的事项：

1. 优化用户注册流程
2. 重构支付模块
3. 开发监控面板
4. 修复上版本遗留 bug

输入：这个方案好像有两个问题吧，一个是性能不够好，还有一个是扩展性比较差
输出：这个方案有两个问题：性能不够好，且扩展性较差。

输入：上周的进展跟大家同步一下，数据库迁移完成了，然后新版首页上线了，还有就是发现了一个支付回调的 bug 已经在修了，另外下周计划是把用户中心重构掉
输出：上周进展同步：

1. [已完成] 数据库迁移完成
2. [已完成] 新版首页上线
3. [进行中] 修复支付回调 bug
4. [下周计划] 重构用户中心
