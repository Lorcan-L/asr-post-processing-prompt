# ASR Prompt 迭代编排方法论 — 交接文档

## 一、方法概述

通过多 Agent 编排进行 prompt 迭代优化。每轮迭代由 4 个独立 Agent 完成，避免上下文污染：

```
Agent A（重写者）→ Agent B（出题者）→ Agent C（测试者）→ Agent D（评审者）
        ↑                                                          |
        └──────────── 下一轮优化方向 ←─────────────────────────────┘
```

## 二、每轮迭代流程

### Agent A：重写者

- **输入**：当前 prompt + 上轮评审报告（问题模式 + 优化方向）
- **输出**：完整的新版 prompt
- **约束**：不做测试，只改 prompt

### Agent B：出题者

- **输入**：本轮侧重方向（不给当前 prompt，避免出"刚好能过"的简单题）
- **输出**：10 条困难测试语料 + 理想输出
- **约束**：语料必须贴近真实场景，每条标注侧重的问题类型

### Agent C：测试者

- **输入**：Agent A 的新 prompt + Agent B 的语料
- **输出**：10 条处理结果
- **约束**：严格按 prompt 行事，模拟真实 LLM 行为

### Agent D：评审者

- **输入**：Agent C 的实际输出 + Agent B 的理想输出
- **输出**：逐条打分（1-5）+ 总分 + 问题模式总结 + 下一轮优化方向
- **约束**：不改 prompt，只评审

### 效率优化

- Agent A + B 可**并行**启动（互不依赖）
- Agent C 依赖 A+B，Agent D 依赖 C
- 后期可将 C+D 合并为一个 Agent 节省轮次
- R8-R10 可合并为"微调+回归测试"一次性完成

## 三、10 轮迭代实际结果

| 轮次  | 得分      | 侧重           | 关键动作                       |
| ----- | --------- | -------------- | ------------------------------ |
| R1    | 34/50     | 精简重构       | 11 条规则 → 5 条 + 8 示例      |
| R2    | **41/50** | 结构化         | 弱化词规则、数字规则、列表触发 |
| R3    | 35/50     | 去重与语序     | 语料难度升级暴露问题           |
| R4    | 32/50     | 边界情况       | 暴露过度书面化                 |
| R5    | 34/50     | 综合测试       | 语域锁定三条铁律               |
| R6    | 32/50     | 跨模型一致     | 7 规则 + 20 示例重构           |
| R7    | **47/50** | 因果+去重+标签 | 关键示例前置，示例驱动         |
| R8-10 | **75/75** | 回归测试       | 微调 2 句话，满分通过          |

### 关键突破点

- **R6→R7（32→47）**：规则从 11→7 条精简 + 示例从 14→23 个按能力分组前置。证明**示例驱动 > 规则驱动**。
- **R4→R5（32→34）**：加入"语域锁定"三条铁律，解决过度书面化。

## 四、核心发现（可复用洞察）

### 1. 示例 > 规则

规则描述越长，弱模型越容易忽略。示例是跨模型最稳定的教学方式。**每条规则至少配 2 个示例**。

### 2. 关键能力前置

LLM 对 prompt 开头的注意力最强（attention 衰减效应）。把最难学的能力（结构化、去重、语序重组）的示例放在最前面。

### 3. 规则数量 ≤ 7

认知负荷理论。11 条规则时弱模型只执行前 3 条；7 条规则时执行率显著提升。

### 4. 语域锁定是必须的

不加语域锁定，模型会把所有输出都往"最正式"的方向拉。必须明确告诉它"日常对话保持口语感"。

### 5. 语料难度要逐步升级

前几轮用简单语料建立基线，后面逐步引入边界情况（极短输入、纯英文、极端去重等）。如果一开始就用最难的语料，prompt 优化会陷入局部最优。

### 6. 分数下降不代表退步

R3（35）< R2（41），但 R3 的语料难度远高于 R2。需要区分"prompt 退步"和"语料升级"。

## 五、可复用场景

此编排方法适用于任何**需要迭代优化 prompt 质量**的场景：

| 场景                 | 适配说明                                               |
| -------------------- | ------------------------------------------------------ |
| **翻译 prompt**      | Agent B 出双语语料，Agent D 按翻译质量标准评审         |
| **客服话术 prompt**  | Agent B 出客户投诉场景，Agent D 按满意度/准确性评审    |
| **代码生成 prompt**  | Agent B 出编程题，Agent C 生成代码，Agent D 判断正确性 |
| **摘要/总结 prompt** | Agent B 出长文，Agent D 按信息保留率/压缩比评审        |
| **数据提取 prompt**  | Agent B 出非结构化文本，Agent D 按提取准确率评审       |
| **风格迁移 prompt**  | Agent B 出原文，Agent D 按风格一致性评审               |

### 复用时需要调整的部分

1. **Agent B 的出题策略**：根据目标场景设计语料分布
2. **Agent D 的评分标准**：根据目标场景定义 1-5 分的标准
3. **迭代侧重安排**：前 2 轮建基线，中间轮针对性强化，最后 2 轮回归测试

### 不需要调整的部分

- 4 Agent 分工架构
- A+B 并行 → C → D 的执行顺序
- "问题模式总结 → 下一轮优化方向"的反馈闭环
- 后期 C+D 合并的效率优化

## 七、后续优化建议

1. **长文本处理**：当前未针对 5 分钟以上语音优化，可在软件层面做分段送入
2. **跨模型适配**：如果切换模型效果下降，优先增加示例而非增加规则
3. **持续迭代**：每积累 5-10 个真实"处理不好"的案例后，跑一轮 Agent 编排迭代
